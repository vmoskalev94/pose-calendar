## 4. Запуск в режиме разработки

### 4.1. Предварительные требования

- Java 21
- Node.js (рекомендуется LTS)
- Docker (для проверки prod-конфигурации опционально)

### 4.2. Backend (dev-профиль, H2)

По умолчанию активен профиль `dev` (H2 в файловом режиме).

```bash
# из корня проекта
./gradlew bootRun

```

Приложение поднимется на:

```
http://localhost:8080

```

Health-check:

```
GET http://localhost:8080/api/health
→ {"status":"OK"}

```

**Конфигурация dev-профиля** — `src/main/resources/application-dev.yml`:

- БД: `jdbc:h2:file:./data/posecalendar_dev;MODE=PostgreSQL;...`
- Flyway включён, миграции лежат в `classpath:db/migration`.

### 4.3. H2 Console (опционально)

Консоль доступна по адресу:

```
http://localhost:8080/h2-console

```

Параметры подключения:

- JDBC URL:

  `jdbc:h2:file:./data/posecalendar_dev;MODE=PostgreSQL;DATABASE_TO_LOWER=TRUE;AUTO_SERVER=TRUE`

- User: `sa`
- Password: пусто

Таблица `app_info` создаётся миграцией `V1__init_app_info.sql`.

---

### 4.4. Frontend (Vite dev server)

```bash
cd frontend
npm install        # первый запуск
npm run dev

```

По умолчанию фронтенд доступен на:

```
http://localhost:5173

```

На странице:

- Заголовок **Pose Calendar**.
- Заглушка блока «Календарь релизов».
- Блок «Статус backend».
- Кнопка **«Проверить backend»**, выполняющая запрос к `/api/health`.

### Proxy для API

В `frontend/vite.config.ts` настроен proxy:

```tsx
server: {
  port: 5173,
  proxy: {
    "/api": {
      target: "http://localhost:8080",
      changeOrigin: true,
    },
  },
},

```

Благодаря этому фронтенд может делать запросы к `/api/...` без указания полного URL.

## 5. Запуск через Docker Compose (prod-профиль, PostgreSQL)

### 5.1. Сборка jar

```bash
./gradlew clean bootJar

```

В результате будет собран jar в `build/libs/*.jar`.

### 5.2. Сборка и запуск контейнеров

```bash
docker compose up --build

```

Запустятся два сервиса:

- `db` — PostgreSQL 16 (образ `postgres:16-alpine`).
- `app` — Spring Boot приложение (профиль `prod`).

Конфигурация `prod` берётся из `application-prod.yml` + env-переменных:

```yaml
spring.datasource.url: jdbc:postgresql://db:5432/posecalendar
spring.datasource.username: posecalendar
spring.datasource.password: posecalendar

```

### 5.3. Проверка

Health-check приложения в Docker:

```
GET http://localhost:8080/api/health
→ {"status":"OK"}

```

## 6. Профили Spring

- **dev** (по умолчанию)
    - H2 (file mode, совместимость с PostgreSQL).
    - Миграции Flyway применяются при старте.
    - Удобен для локальной разработки без Docker.
- **prod**
    - PostgreSQL.
    - Используется при запуске в Docker Compose (через `SPRING_PROFILES_ACTIVE=prod`).

## 7. Flyway миграции

Каталог миграций: `src/main/resources/db/migration`.

Первая миграция:

```sql
V1__init_app_info.sql

CREATE TABLE IF NOT EXISTS app_info (
    id        BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    app_key   VARCHAR(100) NOT NULL UNIQUE,
    app_value VARCHAR(255) NOT NULL
);

INSERT INTO app_info (app_key, app_value)
VALUES ('db.version', 'V1__init_app_info');

```
